Boxbi Messenger

A Comprehensive Journey into Building a Modern Real-Time Chat Application

Final Year Project • 2025-2026

Project Overview

Boxbi Messenger is a full-stack, real-time messaging platform designed to solve the latency issues of traditional HTTP-based communication. Built with a Spring Boot backend and a lightweight Vanilla JS frontend, it leverages WebSockets for instant, bidirectional communication. The application is deployed globally using cloud-native technologies, ensuring reliability and scalability.

Development Journey

Phase 1: Inception & Prototyping

Core WebSocket Implementation

Started by establishing a basic WebSocket connection using Spring Boot's STOMP broker. The goal was simple: send a message from Client A and have Client B receive it instantly.

Phase 2: Data Persistence

Database Integration

Transitioned from in-memory storage to a robust SQL database. Integrated PostgreSQL (prod) and H2 (dev) using Spring Data JPA to store user profiles permanently.

Phase 3: Security Hardening

Authentication & Security

Implemented a comprehensive security layer:

  • BCrypt for password hashing.
  • JWT (JSON Web Tokens) for stateless session management.
  • Input Sanitization to prevent XSS attacks.
Phase 4: Optimization

Automated Maintenance

Developed an automated scheduler (UserCleanupScheduler) to maintain database health by removing inactive users every 24 hours, ensuring optimal performance.

Phase 5: Global Deployment

Going Live

Dockerized the application and deployed the backend to Render.com. The frontend was deployed to Netlify with a custom domain (boxbi.online), making the project accessible worldwide.

Challenges & Solutions

⚠️ Challenge: Security Vulnerabilities

Problem: Early versions used deprecated JWT libraries and plain text passwords.

Solution: Upgraded to the latest JJWT 0.12.3 API and implemented BCrypt encoding. Added extensive input validation to sanitize user inputs against injection attacks.

⚠️ Challenge: Cross-Origin Resource Sharing (CORS)

Problem: When separating frontend and backend on different domains, browsers blocked the connection.

Solution: Configured detailed CORS policies in Spring Security to explicitly allow requests from boxbi.online and vercel/netlify domains.

⚠️ Challenge: Cloud Build Failures

Problem: Render failed to detect the correct Java version and build path.

Solution: Created a custom multi-stage Dockerfile that explicitly controls the build environment, ensuring consistency between local dev and production.
